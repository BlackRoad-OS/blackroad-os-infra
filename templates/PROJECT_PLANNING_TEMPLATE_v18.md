## Usage Guide

### Overview

This template helps track **7 issues** across a project, with **7 signals** for each issue:
1. **Severity** - How critical is the issue?
2. **Impact** - How many users/systems are affected?
3. **Urgency** - How quickly does this need attention?
4. **Owner** - Who is responsible for fixing it?
5. **Fix** - Progress on implementing the fix
6. **Test** - Progress on testing the fix
7. **Ship** - Progress on deploying/releasing the fix

## Best Practices

### When to Use This Template

- **Sprint Planning** - Track multiple issues across a sprint
- **Incident Triage** - Prioritize and track multiple concurrent incidents
- **Project Status** - Weekly/daily status updates for stakeholders
- **Release Planning** - Track blockers and readiness across issues
- **Team Standup** - Visual dashboard for team discussions

### Tips for Effective Usage

1. **Update Regularly** - Keep the status current (daily or at each standup)
2. **Be Honest** - Use red/yellow indicators truthfully to surface problems early
3. **Add Context** - Use mood emojis (ğŸ˜­ğŸ˜°ğŸ˜ğŸ™‚ğŸ‰) to communicate urgency and sentiment
4. **Assign Owners** - Every issue should have a clear owner
5. **Track Progress** - Update Fix/Test/Ship signals as work progresses
6. **Limit Scope** - If you have more than 7 issues, consider creating multiple templates or prioritizing

### Integration with Other Tools

This template complements:
- GitHub Issues (reference issue numbers in the template)
- Project boards (sync status with board columns)
- Slack/Discord updates (paste template for quick team visibility)
- Weekly reports (snapshot of project health)
